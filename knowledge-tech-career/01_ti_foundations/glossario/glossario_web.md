# Glossário web e HTTP

Ordem alfabética.

- **AJAX** — técnica que usa JavaScript para trocar dados com o servidor sem recarregar a página.
- **API** — interface de programação que expõe funcionalidades de um software a outro.
- **Autenticação** — ato de provar identidade (senha, token, certificado).
- **Autorização** — ato de verificar se identidade autenticada pode acessar recurso específico.
- **Bearer token** — token portado no cabeçalho `Authorization` que autoriza o portador.
- **Body (corpo)** — parte da requisição ou resposta HTTP que carrega dados.
- **Browser** — software cliente que renderiza páginas web.
- **Cookie** — pequeno dado armazenado pelo navegador por solicitação do servidor.
- **CORS** — política que controla quais origens podem acessar recursos de outra origem.
- **CSS** — linguagem que define aparência de documentos HTML.
- **CSRF** — ataque que explora sessão autenticada para executar ação indesejada.
- **DOM** — representação em árvore do documento HTML em memória no navegador.
- **Endpoint** — URL específica de uma API.
- **Fetch** — API JavaScript moderna para requisições HTTP no navegador.
- **GET** — método HTTP que lê recurso sem alterar estado.
- **Header (cabeçalho)** — campo de metadados em requisição ou resposta HTTP.
- **HTML** — linguagem de marcação que define estrutura e semântica do conteúdo web.
- **HTTP** — protocolo de aplicação da web, cliente-servidor, stateless.
- **HTTPS** — HTTP sobre TLS; criptografado, autenticado, íntegro.
- **HTTP/2, HTTP/3** — versões que multiplexam, comprimem headers e reduzem latência.
- **JSON** — formato textual de troca de dados estruturados.
- **JWT** — *JSON Web Token*; token assinado usado para autenticação stateless.
- **Markup** — marcação de conteúdo com tags, como em HTML e XML.
- **MIME type** — identificador de tipo de conteúdo (`text/html`, `application/json`).
- **OAuth** — protocolo de autorização delegada usado para login com Google, GitHub etc.
- **Origin** — esquema + host + porta de uma URL; base da política de mesma origem.
- **POST** — método HTTP que cria recurso no servidor.
- **Query string** — parâmetros após `?` na URL (ex.: `?id=123&sort=data`).
- **REST** — estilo arquitetural de API baseado em recursos e verbos HTTP.
- **SameSite** — atributo de cookie que restringe envio em requisições cross-site.
- **SPA** — aplicação web de página única; navegação sem recarregar o documento.
- **SSR** — renderização do HTML no servidor antes de enviar ao cliente.
- **Status code** — número de 3 dígitos que resume resultado da requisição HTTP.
- **TLS** — protocolo de criptografia da web; componente do HTTPS.
- **URL** — localizador uniforme de recurso; endereço completo (esquema, host, path, query).
- **User-Agent** — cabeçalho que identifica o cliente (navegador, versão, SO).
- **WebSocket** — protocolo full-duplex sobre TCP para comunicação em tempo real.
- **XHR** — *XMLHttpRequest*; API antiga para AJAX, antecessora do fetch.
- **XML** — formato de marcação estruturada usado em documentos formais e legados.
- **XSS** — ataque que injeta script malicioso em página confiável.
