# Glossário básico de TI

Termos gerais em ordem alfabética. Cada verbete: definição em 1 frase. Tag `[bloco]` indica onde aprofundar.

- **Algoritmo** — sequência finita de passos que resolve um problema. `[software_lifecycle]`
- **API** — interface pela qual um software expõe funções para outro consumir. `[internet_web_basics]`
- **Argumento** — valor passado para uma função ou comando. `[concepts]`
- **Arquivo** — conjunto de bytes nomeado e persistido em disco. `[hardware_networks_os]`
- **ASCII** — tabela de codificação de 128 caracteres em 7 bits. `[concepts]`
- **Backup** — cópia de dados armazenada em local separado para recuperação após falha. `[software_lifecycle]`
- **Banco de dados** — coleção organizada de dados acessada por consultas. `[concepts]`
- **Base64** — codificação de dados binários em caracteres ASCII, aumentando tamanho em ~33%. `[concepts]`
- **Binário** — sistema numérico base 2; também chamado executável compilado. `[concepts]`
- **Bit** — menor unidade de informação, 0 ou 1. `[concepts]`
- **Byte** — 8 bits; unidade mínima endereçável de memória. `[concepts]`
- **Bug** — defeito em software. `[software_lifecycle]`
- **Cache** — armazenamento rápido e temporário de dados frequentemente acessados. `[hardware_networks_os]`
- **CI/CD** — integração e entrega contínuas; pipeline automatizado de build, teste e deploy. `[software_lifecycle]`
- **Cliente** — software que inicia requisição a um servidor. `[concepts]`
- **CLI** — interface de linha de comando, sem janelas gráficas. `[concepts]`
- **Cloud (nuvem)** — recursos computacionais consumidos sob demanda de um provedor. `[concepts]`
- **Commit** — registro atômico de mudança no controle de versão. `[software_lifecycle]`
- **Compilador** — programa que traduz código-fonte em binário executável. `[software_lifecycle]`
- **CPU** — unidade central de processamento; executa instruções. `[hardware_networks_os]`
- **Criptografia** — técnica de codificar informação para que só autorizados a leiam. `[hardware_networks_os]`
- **Debug** — processo de encontrar e corrigir defeito. `[software_lifecycle]`
- **Dependência** — biblioteca ou serviço que um software precisa para funcionar. `[software_lifecycle]`
- **Deploy** — ato de colocar software em ambiente de execução, geralmente produção. `[software_lifecycle]`
- **DevOps** — cultura que integra desenvolvimento e operações. `[software_lifecycle]`
- **DNS** — sistema que traduz nomes de domínio em endereços IP. `[hardware_networks_os]`
- **Docker** — plataforma de contêineres que empacota aplicação e suas dependências. `[software_lifecycle]`
- **Driver** — software que permite ao SO falar com um dispositivo específico. `[hardware_networks_os]`
- **Endpoint** — URL específica de uma API. `[internet_web_basics]`
- **Firmware** — software gravado dentro de um dispositivo de hardware. `[concepts]`
- **Framework** — estrutura de código reutilizável que inverte o controle sobre a aplicação. `[software_lifecycle]`
- **Git** — sistema distribuído de controle de versão. `[software_lifecycle]`
- **Hash** — função que transforma entrada arbitrária em string de tamanho fixo. `[hardware_networks_os]`
- **HTTP** — protocolo da web para requisição e resposta de recursos. `[internet_web_basics]`
- **HTTPS** — HTTP sobre TLS, criptografado e autenticado. `[internet_web_basics]`
- **IDE** — ambiente integrado de desenvolvimento com editor, debugger e build. `[software_lifecycle]`
- **IP** — endereço numérico de dispositivo em rede. `[hardware_networks_os]`
- **JSON** — formato textual leve para troca de dados estruturados. `[internet_web_basics]`
- **Kernel** — núcleo do sistema operacional com acesso privilegiado ao hardware. `[hardware_networks_os]`
- **LAN** — rede local, restrita a casa, escritório ou prédio. `[hardware_networks_os]`
- **Linter** — ferramenta que analisa código e aponta problemas de estilo ou risco. `[software_lifecycle]`
- **Log** — registro cronológico de eventos de um sistema. `[software_lifecycle]`
- **Open source** — software com código-fonte aberto e licença que permite modificá-lo. `[concepts]`
- **Pacote** — unidade distribuível de software com metadados (versão, dependências). `[software_lifecycle]`
- **Patch** — correção aplicada a software já existente; também incremento menor de SemVer. `[software_lifecycle]`
- **Porta** — número que identifica serviço em um host de rede. `[hardware_networks_os]`
- **Processo** — instância em execução de um programa. `[hardware_networks_os]`
- **RAM** — memória volátil de trabalho, perdida ao desligar. `[hardware_networks_os]`
- **Refatoração** — alteração de código que melhora estrutura sem alterar comportamento externo. `[software_lifecycle]`
- **Repositório** — local onde o código e seu histórico são armazenados (Git, SVN). `[software_lifecycle]`
- **Requisito** — funcionalidade ou restrição que o sistema deve atender. `[software_lifecycle]`
- **Script** — programa curto interpretado, geralmente para automação. `[concepts]`
- **SemVer** — convenção de versionamento MAJOR.MINOR.PATCH com semântica definida. `[software_lifecycle]`
- **Servidor** — software que atende requisições de clientes; também máquina física com esse papel. `[concepts]`
- **SSH** — protocolo seguro para acesso remoto a máquina por linha de comando. `[hardware_networks_os]`
- **SSD** — armazenamento não volátil baseado em chips flash. `[hardware_networks_os]`
- **Syscall** — chamada de sistema; forma como aplicação pede serviço ao kernel. `[hardware_networks_os]`
- **TCP** — protocolo de transporte confiável com ordem e retransmissão. `[hardware_networks_os]`
- **Thread** — fluxo de execução dentro de um processo; várias threads compartilham memória. `[hardware_networks_os]`
- **TLS** — protocolo que criptografa e autentica conexões sobre TCP. `[hardware_networks_os]`
- **UDP** — protocolo de transporte sem conexão e sem garantia de entrega. `[hardware_networks_os]`
- **UTF-8** — codificação de texto Unicode compatível com ASCII, 1 a 4 bytes por caractere. `[concepts]`
- **Variável de ambiente** — valor nomeado disponível ao processo, configurável fora do código. `[software_lifecycle]`
- **VPN** — rede privada virtual que estende rede interna por túnel criptografado. `[hardware_networks_os]`
- **Webhook** — URL pública que recebe notificação HTTP quando evento ocorre em outro sistema. `[internet_web_basics]`
- **YAML** — formato de serialização indentado, usado para configuração legível. `[internet_web_basics]`
