# Panorama de Carreira em TI

## O que é TI

TI (Tecnologia da Informação) não é uma profissão única. É um guarda-chuva que cobre desde quem troca cabo de rede até quem treina modelo de IA em cluster GPU. Dizer "trabalho com TI" tem o mesmo grau de precisão que dizer "trabalho com saúde" — sem especificar se é enfermeiro, neurocirurgião ou perito.

A consequência prática: planejar carreira em TI exige primeiro escolher a **família** (grande área), depois o **papel** dentro da família, depois o **nível de senioridade**. Pular essa decomposição gera frustração (estudar coisa errada) e tempo perdido.

## Grandes famílias

| Família | Foco central | Exemplos de papéis |
|---|---|---|
| Suporte / Infra | Manter sistemas vivos, atender usuário | Helpdesk, analista de suporte, sysadmin |
| Redes | Conectar máquinas, roteamento, firewall | Analista de rede, engenheiro de rede, arquiteto de rede |
| Desenvolvimento | Construir software | Dev front/back/full, engenheiro de software, mobile |
| Dados | Extrair, modelar, analisar informação | Analista de dados, cientista de dados, engenheiro de dados |
| Segurança | Proteger sistemas e dados | Analista SOC, pentester, CISO, engenheiro de AppSec |
| Produto | Traduzir negócio em software | PM, PO, product analyst |
| Design / UX | Experiência e interface | UX researcher, UI designer, product designer |
| Arquitetura | Desenhar sistemas grandes | Arquiteto de software, de soluções, corporativo |
| IA / ML | Modelos que aprendem | Engenheiro de ML, MLOps, engenheiro de IA aplicada |
| DevOps / SRE | Automação, deploy, confiabilidade | DevOps, SRE, platform engineer |

## Como um médico perito pode transitar

O perfil do Dr. Jesus já carrega três ativos que encurtam caminho:

1. **Domínio médico formal + CRM ativo** — vale ouro em qualquer interseção saúde/TI.
2. **Experiência pericial** — pensamento estruturado, decomposição de evidência, produção de documento técnico: isomorfo ao raciocínio de analista/cientista.
3. **Ecossistema Dexter já em produção** — automação, Python, hooks, MCP, Claude Code: não parte do zero.

Trilhas naturais (maior aderência → menor):

- Informática médica / health-tech (ponte direta).
- Cientista de dados em saúde, RWE, apoio à decisão clínica.
- Engenheiro de IA aplicada a laudos, triagem, pericial.
- Product manager de produto médico/jurídico.
- Especialista em segurança/LGPD em saúde.

Trilhas de baixa aderência (custo alto, retorno baixo para este perfil):

- Front-end puro.
- Sysadmin clássico.
- Redes baixo nível.

## Erros comuns de transição

- Querer "aprender TI" sem escolher família → estudo disperso, nada rende.
- Copiar trilha de dev recém-formado (bootcamp front-end) → ignora ativo médico.
- Acumular certificação sem entregar projeto → CV vazio de prática.
- Separar "medicina" de "TI" como mundos distintos → perde diferencial de interseção.
